Angular Lifecycle Events

ngOnChanges

Called before ngOnInit() and whenever one or more data-bound input properties change.

Scenario: Update a chart when input data changes.

Example:


<script>
ngOnChanges(changes: SimpleChanges): void {
  if (changes['data']) {
    this.updateChart(changes['data'].currentValue);
  }
}
</script>
ngOnInit

Called once, after the first ngOnChanges(). Initializes the component.

Scenario: Fetch initial data from a server.

Example:


<script>
ngOnInit(): void {
  this.fetchData();
}
</script>
ngDoCheck

Called during every change detection run, allows custom change detection logic.

Scenario: Detect changes that Angular doesn't catch.

Example:


<script>
ngDoCheck(): void {
  if (this.hasChanges()) {
    this.updateView();
  }
}
</script>
ngAfterContentInit

Called once after the component's content has been initialized.

Scenario: Initialize content projection.

Example:


<script>
ngAfterContentInit(): void {
  this.initProjectedContent();
}
</script>
ngAfterContentChecked

Called after the component's content has been checked by the change detection mechanism.

Scenario: React to projected content changes.

Example:


<script>
ngAfterContentChecked(): void {
  this.checkProjectedContent();
}
</script>
ngAfterViewInit

Called once after the component's views and child views have been initialized.

Scenario: Access child components or view children.

Example:


<script>
ngAfterViewInit(): void {
  this.initChildComponents();
}
</script>
ngAfterViewChecked

Called after the component's views and child views have been checked by the change detection mechanism.

Scenario: Perform actions after view updates.

Example:


<script>
ngAfterViewChecked(): void {
  this.verifyViewChanges();
}
</script>
ngOnDestroy

Called once before the component is destroyed. Use for cleanup logic.

Scenario: Unsubscribe from observables.

Example:


<script>
ngOnDestroy(): void {
  this.subscription.unsubscribe();
}
</script>
